# 데이터베이스 보안 가이드 (DB Security)

1. [SQL 인젝션] 모든 DB 쿼리는 Parameterized Query (Prepared Statement)를 사용해야 합니다. 문자열 더하기 방식은 엄격히 금지합니다.
2. [계정 관리] DB 연결 시 root/admin 계정 대신 최소 권한을 가진 계정을 사용하세요.
3. [비밀번호 암호화] 사용자 비밀번호는 PBKDF2, Argon2 등 강력한 해시 알고리즘을 사용하되 반드시 Salt를 가미해야 합니다.
4. [설정 파일 노출] `.env`나 설정 파일에 DB 접속 정보가 하드코딩되어 있지 않은지 확인하세요.
